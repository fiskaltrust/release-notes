# InStore App 1.3.0
[![Static Badge](https://img.shields.io/badge/milestone-v1.3.0-green?logo=github)](https://github.com/fiskaltrust/fiskaltrust-instore-app/milestones?state=closed)

## Payment vendor provided tip support across providers

Tips entered directly on the payment terminal are now reported back to the POS in a consistent way across Viva, GP Pay, GP tom and Shift4 / S4.

_Affected issues:_  
[Payment] tip handling - payment provider -> POS - all payment providers support it similar to how it is done in the dummy provider [#534](https://github.com/fiskaltrust/fiskaltrust-instore-app/issues/534)  
[Payment] Viva - add support for payment vendor provided TIP [#626](https://github.com/fiskaltrust/fiskaltrust-instore-app/issues/626)  
[Payment] GP Pay - payment vendor provided tip support [#632](https://github.com/fiskaltrust/fiskaltrust-instore-app/issues/632)  
[Payment] GPtom - add payment vendor provided TIP [#630](https://github.com/fiskaltrust/fiskaltrust-instore-app/issues/630)  
[Payment] Shift4 - support report of tip added by the user through the payment vendor app/interface [#384](https://github.com/fiskaltrust/fiskaltrust-instore-app/issues/384)

## Refund and cancel support expanded across payment providers

Refund and cancel transactions - including payment vendor provided tip - are now supported on hobex ECR, GP Pay, GP tom, Hobex POSit, Softpay.io and PayOne (Tab on Mobile).

_Affected issues:_  
[Payment] hobex ECR - support refund and cancel including payment vendor provided tip [#620](https://github.com/fiskaltrust/fiskaltrust-instore-app/issues/620)  
[Payment] GP Pay - refund + cancel support [#635](https://github.com/fiskaltrust/fiskaltrust-instore-app/issues/635)  
[Payment] GP tom - support for refund + cancel transactions [#633](https://github.com/fiskaltrust/fiskaltrust-instore-app/issues/633)  
[Payment] Hobex POSit and Softpay.io - Refund + cancel support including payment vendor based tip [#609](https://github.com/fiskaltrust/fiskaltrust-instore-app/issues/609)  
[Payment] PayOne - Tab on Mobile - finalisieren (payment, refund, cancel, tip) [#585](https://github.com/fiskaltrust/fiskaltrust-instore-app/issues/585)
